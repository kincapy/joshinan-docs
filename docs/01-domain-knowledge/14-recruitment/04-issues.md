# 募集業務の課題と解決策

募集業務において現在発生している課題と、それに対する解決策のアイデアをまとめる。

## 課題

### 課題①：エージェントから届く書類の質がまばら

定められている提出すべき書類が揃っていない。不足がある、訳文が抜けているといった問題が頻繁に発生する。

- 提出すべき書類が揃っていない・不足がある
- 原文に対する訳文が抜けている

### 課題②：エージェント自体が必要書類を理解していない

何の書類を出すべきかをエージェント自身が把握できていないケースがある。

具体例：
- 経費支弁書を出す際に、経費支弁者と本人の関係性を立証する資料（出生証明書・親族関係公証書等）が欠落
- ノータリーオフィス（公証役場）による公証が必要な書類に公証がない
- 残高証明書に付随すべき所得証明（納税証明書・源泉徴収票に相当する書類）が未添付
- 経費支弁者の資産形成能力を立証する資料の選び方を理解していない

### 課題③：学校側で受け取った書類が何の書類か分からない

原文に訳文が付いていないと、紙をまとめて渡されるだけになり、学校側でそれが何の書類なのかを把握できない。

- 原文が何の書類か分からない
- 書類の順番がまばらで、正しい順番に並び替えることができない
- 原文がどういう内容かを確認するためには、その国の言語が分かる人員が必要になる

### 課題④：履歴書から申請書への転記作業

エージェントから学生の履歴書をもらい、それを元に在留資格認定証明書交付申請書に転記する作業がある。現在は手作業で行っており、転記ミスや作業負荷が課題。

**理想的な状態**: エージェントから履歴書が届いた瞬間に、在留資格認定証明書交付申請書が自動生成される。

## 解決策アイデア

### アップロードフォーム + AI による書類管理

#### 1. エージェント向けアップロードフォームの提供

日本語学校からエージェントに対して、学生ごとのアップロードフォームを提供する。エージェントはこのフォームを通じて、学生ごとにファイルをアップロードしていく。

#### 2. AIを活用した自動チェック

アップロードされた書類をAI（Claude API等）で解析し、以下を自動チェックする。

| チェック項目 | 内容 |
|------------|------|
| 書類の網羅性 | 定められている書類がすべて揃っているか |
| 訳文の有無 | 原文に対する日本語訳が付いているか |
| 書類の並び順 | 入管提出の正しい順番に並んでいるか |

#### 3. 不足書類の可視化

学生ごと（＝ケース単位）で、何の書類が足りないかを一覧表として表示する。エージェント側・学校側の双方から確認できる状態にすることで、やり取りの回数を減らす。

::: tip 将来的なシステム化の展望
この解決策は、[将来構想のチャットボット](/02-system-specification/00-concept/)とも連携可能。書類管理のデータ基盤を整備することで、募集業務のDXを段階的に進められる。
:::
