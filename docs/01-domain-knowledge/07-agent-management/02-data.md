# データ — エージェント正規化・識別子・費用

## エージェント名 正規化ルール

送出機関の名称は、請求書・学生データ・契約書などで表記が異なるため、以下のルールで正規化します。

### 正規化のパターン

| パターン | ルール | 例 |
|---------|--------|-----|
| グループ企業 | 複数法人が同一グループに属する場合、グループ名に統一する | 法人A・法人B → グループ名 |
| 個人名エージェント | 国名をプレフィックスとして付与し、一意に識別する | `国名_個人名` |
| 系列企業 | ブランド名ごとに区別して管理する | 系列A、系列B |

## 識別子

| ID | 形式 | 用途 |
|----|------|------|
| 学籍番号 | 7桁数字（例: 2504006） | 学生の一意識別。全シート共通キー |
| 請求書ID | INV-XXX | エージェント請求書の識別 |
| 支払ID | PAY-XXX | エージェント支払いの識別 |

## 学生名簿上のエージェント管理項目

エージェントへの支払い管理のために、学生名簿上に以下の情報が必要。

| 項目 | 内容 | 備考 |
|------|------|------|
| エージェント名 | どのエージェント経由の学生か | 正規化後の名称で記録 |
| エージェント費用支払済フラグ | この学生のエージェント費用を支払ったか否か | `true` / `false` |

::: info なぜ学生単位で管理するか
エージェント費用は「学生1名あたりいくら」で発生する。学生名簿上にフラグを持つことで、未払いの学生を一覧で把握し、請求書との突合ができる。
:::

## エージェント費用の構造

- エージェント費用は学生1名あたりの紹介手数料として発生する
- 請求書の法人名と正規化後の名称が異なる場合があるため、照合時は正規化ルールを必ず適用すること
- グループ企業の場合、複数法人を経由した請求が発生するため、グループ単位での債務管理が必要
