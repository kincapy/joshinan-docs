# 課題

学生管理における現在の主要な課題を整理します。

## 1. Ringualの使いづらさ

### 課題

学校管理システム「Ringual」のUIがプロンプトベースで直感的でなく、日常業務での操作効率が低い。

### 背景

- Ringualは日本語学校向けの管理システムとして導入されているが、操作体系がプロンプト（コマンド入力）ベースに近い設計になっている
- 一覧画面から詳細への遷移、検索、フィルタリングといった基本操作が煩雑
- エクスポート機能はあるが、出力されるカラム順や日本語カラム名に表記揺れがあり、後続処理で手間がかかる
- スタッフ全員がITに精通しているわけではなく、操作に慣れるまでの学習コストが高い

### ポイント

- **プロンプトベースUI** のため、「何をどう入力すればよいか」が分かりにくい
- 結果として、Ringualを使いこなせるスタッフが限られ、業務が属人化する一因になっている
- データのエクスポート → 加工 → 活用 の流れにおいて、Ringualからの出力品質が安定しないことがボトルネック

::: tip 改善の方向性
Ringualのデータを定期的にエクスポートし、より使いやすい形式（スプレッドシートやダッシュボード）で閲覧・編集できる仕組みを構築することで、Ringualの操作頻度を最小限に抑えるアプローチが現実的です。
:::

## 2. データの属人化

### 課題

学生に関するデータが複数の場所に分散しており、特定のスタッフしかアクセス・理解できない状態になっている。

### 背景

- 学生データの保管場所が統一されておらず、以下のように分散している：
  - **Box** — 書類のスキャンデータ、契約書PDF
  - **Google Drive** — 各担当者が個別に作成した管理シート
  - **Ringual** — 学籍情報の正式なマスターデータ
  - **Excel（ローカル）** — スタッフが個人PCで管理している補足データ
- 「この情報はどこにあるか」「最新版はどれか」が分かりにくい
- 担当者が不在のとき、必要な情報にアクセスできないケースが発生している

### ポイント

- **データの散在**：同じ学生の情報が4つ以上の場所に分かれている
- **バージョン管理の不在**：どのファイルが最新か分からない
- **引き継ぎの困難**：担当者が異動・退職した際に、データの場所やルールが分からなくなるリスクが高い

::: warning リスク
データの属人化は、入管報告や監査対応の際に「データが見つからない」「最新データか確認できない」という問題に直結します。法令対応に支障をきたす前に、データの一元化を進める必要があります。
:::

## 3. 面談記録の入力効率

### 課題

学生との面談内容を記録する際、手動入力に時間がかかり、記録が後回しになったり不完全になったりしている。

### 背景

- 学生との面談（進路相談、出席指導、生活指導など）は頻繁に発生する
- 現状では、面談後にスタッフがExcelやWordに手動で記録を入力している
- 面談中にメモを取り、後からまとめて入力する運用のため、入力漏れや記憶違いが生じやすい
- 面談記録は入管への報告や監査対応で求められることがあり、正確性が重要

### ポイント

- **音声入力 → Claude API → DB自動登録** の仕組みで入力効率を大幅に改善できる可能性がある
- 具体的には：面談中にスマートフォンで音声録音 → 音声をテキスト化 → Claude APIで構造化（日付、学生名、面談種別、内容要約、アクション項目を抽出） → データベースに自動登録
- 手動入力の手間を削減しつつ、記録の網羅性と正確性を向上させる

::: tip 実現に向けて
音声入力による面談記録の自動化は、以下のステップで段階的に導入できます：
1. まずは音声録音 → テキスト化（Whisper API等）の仕組みを検証
2. Claude APIで面談内容を構造化するプロンプトを設計
3. 構造化データをデータベースに登録するAPIを実装
4. スタッフにスマートフォンアプリとして提供
:::
