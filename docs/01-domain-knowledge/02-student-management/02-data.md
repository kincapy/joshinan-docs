# データ — 学生マスター（126カラム）

## 学生マスターデータ

学生マスターは126カラムで構成される、学生管理の中核データです。以下にカテゴリごとの主要カラムをまとめます。

| カテゴリ | 主要カラム | 備考 |
|----------|-----------|------|
| 基本情報 | 学籍番号（7桁）、氏名（漢字・カナ・英語）、生年月日、性別、国籍 | 学籍番号は入学年＋コホート＋連番で構成 |
| 学籍情報 | ステータス、入学年月日、卒業予定日、クラス、コホート | ステータス：下記参照 |
| 連絡先 | 電話番号、メールアドレス、日本の住所、母国の住所 | 緊急連絡先（母国の家族）も含む |
| 旅券・在留 | パスポート番号、在留カード番号、在留資格、在留期限 | 在留期限の管理は入管対応の基本 |
| 入国情報 | 入国日、入国空港、便名、出迎え担当 | 空港出迎えの手配に使用 |
| 資格外活動 | 資格外活動許可の有無、許可日、許可期限 | アルバイト管理の前提情報 |
| 健康保険 | 国民健康保険番号、加入日、保険証番号 | 病院付き添い時に必要 |
| エージェント | エージェント名、エージェントコード、担当者名 | 正規化済み・上書き禁止（後述） |
| 勤務先（1〜3） | 勤務先名、電話番号、勤務時間、時給 | 最大3つまで登録可能。週28時間の管理に使用 |
| 卒業後の予定 | 進路区分（進学/就職/帰国）、進路先名、合否 | 在学中から段階的に入力 |

## 学生ステータスの内部コード

RINGUALでは学生ステータスを以下の内部コードで管理しています。システム連携時にはこのコードが使用されます。

| 内部コード | ステータス | 備考 |
|-----------|-----------|------|
| -10 | 未登録 | 初期状態 |
| 0 | 入学前 | COE申請中など。詳細ステータス（申請中/入国待ち/不交付）あり |
| 1 | 在学 | 通常の在籍状態 |
| 51 | 休学 | — |
| 81 | 退学 | 退学年月は上書き禁止（後述） |
| 82 | 除籍 | — |
| 91 | 卒業 | — |
| 92 | 修了 | — |

::: info 入学前ステータスの細分類
入学前（コード0）の学生には、さらに「申請中 / 入国待ち / 不交付」の詳細ステータスがあります。COE申請の進捗管理に使用します。
:::

## データソース

### Ringual からのエクスポート

学生マスターの元データは **Ringual**（学校管理システム）に登録されています。

**エクスポート手順：**

1. Ringual にログイン
2. 「学生管理」→「学生一覧」を開く
3. フィルタで対象コホート・ステータスを絞り込み
4. 「エクスポート」ボタンからExcel形式でダウンロード
5. ダウンロードしたファイルを所定の命名規則でリネーム

::: warning Ringual エクスポートの注意点
- エクスポートされるカラム順が一定でない場合があるため、取り込み時にカラム名で照合すること
- 日本語カラム名に表記揺れがあるため、マッピングテーブルで吸収する必要あり
- 一度にエクスポートできる件数に制限がある場合は、コホート別に分割してエクスポート
:::

## 更新時の保護ルール

学生マスターには、データ整合性を守るために**上書き禁止**のカラムがあります。

| カラム | 保護ルール | 理由 |
|--------|-----------|------|
| 退学年月 | 上書き禁止 | 一度記録された退学日は変更不可。入管報告との整合性を保つため |
| エージェント情報 | 上書き禁止 | 正規化済みのエージェント名・コードは手動での変更を禁止。エージェント管理マスターとの整合性を保つため |

::: danger 上書き禁止カラムの取り扱い
これらのカラムを更新する必要がある場合は、管理者に確認の上、変更履歴を残して対応してください。自動処理で上書きしないよう、インポート処理では必ずスキップロジックを入れてください。
:::

## ファイル命名規則

学生マスターのExcelファイルは、以下の命名規則に従います。

```
学生名簿マスター_YYYYMMDD.xlsx
```

- `YYYYMMDD`：エクスポート日（例：20260209）
- 例：`学生名簿マスター_20260209.xlsx`

::: tip 命名規則の意図
日付をファイル名に含めることで、どの時点のスナップショットかを明確にします。差分管理やバックアップの際にも、ファイル名だけで時系列が把握できます。
:::
