# 業務ナレッジ

常南国際学院の業務に関する知識を15カテゴリで体系化しています。

## カテゴリ一覧

| # | カテゴリ | 内容 | 優先度 |
|---|---------|------|--------|
| 00 | [日本語教育機関](./00-japanese-language-education/) | 業界全体の基礎知識（入管法、告示基準、適正校基準） | 高 |
| 01 | [学校基本情報](./01-school-info/) | 単一校の基本情報 | 低 |
| 02 | [学生管理](./02-student-management/) | 入学〜卒業のライフサイクル管理 | 最高 |
| 03 | [カリキュラム・時間割](./03-curriculum/) | コース→科目→授業の階層 | 中 |
| 04 | [クラス編成](./04-class-assignment/) | 学生→クラス→教室→教員の割当 | 中 |
| 05 | [出席管理](./05-attendance/) | 法定出席率要件の管理 | 高 |
| 06 | [学費管理](./06-tuition/) | 請求・入金・残高の月次サイクル | 最高 |
| 07 | [エージェント管理](./07-agent-management/) | 送出機関・仲介業者の管理 | 最高 |

## ファイル構成パターン

各カテゴリは以下のファイルで構成されます：

| ファイル | 内容 | 記述の粒度 |
|---------|------|-----------|
| `README.md` | カテゴリの概要、リンク、関連カテゴリ | 5〜15行 |
| `01-work-flow.md` | 担当者、目的、業務ステップ、アウトプット | 100〜300行。Mermaidフロー図あり |
| `02-data.md` | オブジェクトの項目・用途・値の例 | 100〜400行。テーブル形式 |
| `03-issues.md` | 課題・問題点（課題→背景→ポイントの3段構成） | 50〜150行 |

## カテゴリ間の相互参照

- 学生管理 ↔ 学費管理（学生ごとの請求・残高）
- 学生管理 ↔ 出席管理（出席率の計算・入管報告）
- 学生管理 ↔ エージェント管理（どのエージェント経由か）
- 学生管理 ↔ 入管報告（在籍状況・出席状況の報告対象）
- 学費管理 ↔ エージェント管理（エージェント費用の請求書）
