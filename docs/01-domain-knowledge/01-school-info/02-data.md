# 学校基本情報 データ定義

学校基本情報のシステム管理に必要なデータオブジェクトを定義します。

## 学校マスタ (School)

学校の基本情報を管理するマスタデータ。1レコード＝1校。常南国際学院では基本的に1校のみだが、将来の多校展開やグループ校管理を見据えた設計。

### 項目

| 項目 | 値の例 | 備考 |
|------|--------|------|
| 学校名 | 常南国際学院 | 正式名称 |
| 学校番号 | 523 | 入管から配布された3桁の番号。ビザ申請書類に記載 |
| 所在地 | 茨城県牛久市○○ | 校舎の住所 |
| 電話番号 | 029-XXX-XXXX | 代表番号 |
| 運営法人名 | ○○株式会社 | 設置者の法人名 |
| 法人種別 | 株式会社 | 学校法人 / 株式会社 / その他 |
| 適正校分類 | クラスI | クラスI（在籍管理優良校）/ クラスII（適正校）/ 非適正校 |
| 告示番号 | 告示○○号 | 法務省告示の番号。認定日本語教育機関への移行後は認定番号に変わる可能性あり |
| 定員数 | 200 | 入管に届け出た収容定員 |
| 設立年月日 | 2018-04-01 | 告示日または設立日 |

::: info 適正校分類について
適正校の判定基準は [00 日本語教育機関 > データ](/01-domain-knowledge/00-japanese-language-education/02-data) を参照。問題在籍率に基づき毎年判定される。
:::

## 入学時期マスタ (EnrollmentPeriod)

年度ごとの入学時期と募集定員を管理するデータ。入学時期は入管制度上4パターンあるが、常南国際学院では4月生・10月生が主力。

### 項目

| 項目 | 値の例 | 備考 |
|------|--------|------|
| 入学月 | 4月 | 4月 / 10月 / 1月 / 7月 |
| 在籍期間（月数） | 24 | 4月生=24ヶ月、10月生=18ヶ月、1月生=15ヶ月、7月生=21ヶ月 |
| 年度 | 2026 | 西暦の年度 |
| 募集定員 | 60 | その入学時期の募集人数。定員×交付率を考慮して設定 |

### 入学月と在籍期間の対応

| 入学月 | 在籍期間 | 授業料総額（目安） | 備考 |
|--------|---------|-------------------|------|
| 4月 | 2年間（24ヶ月） | 約140万円 | 最も収益性が高い。主力 |
| 10月 | 1.5年間（18ヶ月） | 約105万円 | 海外の学期末に合わせやすい |
| 1月 | 1年3ヶ月（15ヶ月） | 約87万円 | 募集校は少ない |
| 7月 | 1年9ヶ月（21ヶ月） | 約122万円 | 募集校は少ない |

::: warning 募集定員の設計
募集定員は単純な空き定員ではなく、ビザ交付率を考慮して設計する必要がある。例: 空き定員50名 × 交付率80% = 実質40名入学。入管制度上「申請可能数は欠員数の1.2倍まで」の制限がある。
:::

## 費用項目マスタ (FeeItem)

学校が徴収する費用の品目定義。学費管理で使用する品目マスタ（[06 学費管理 > データ](/01-domain-knowledge/06-tuition/02-data)）の元となるデータ。

### 項目

| 項目 | 値の例 | 備考 |
|------|--------|------|
| 費用項目名 | 入学金 | 品目の名称 |
| 金額 | 55,000 | 円。税込 |
| 徴収タイミング | 入学時 | 入学時 / 毎月 / 年次 / その他 |

### 主な費用項目

| 費用項目 | 金額（税込） | 徴収タイミング | 備考 |
|----------|------------|---------------|------|
| 入学金 | ¥55,000 | 入学時 | 1回のみ |
| 年間授業料 | ¥693,000 | 毎月（月割） | 月額 ¥69,300 × 10ヶ月（3月・8月除く） |
| 寮費 | ¥25,000 | 毎月 | 相部屋の場合。12ヶ月分 |
| 教材費 | ¥33,000 | 年次 | 1名あたり年間約1.5万〜3.3万 |

::: tip 費用項目の詳細
費用項目の全一覧と請求ロジックについては [06 学費管理 > データ](/01-domain-knowledge/06-tuition/02-data) を参照。
:::
