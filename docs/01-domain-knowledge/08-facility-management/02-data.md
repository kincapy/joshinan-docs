# データ — 物件・WiFi・水光熱費

## 物件管理（寮管理）

### 基本構造

留学生は基本的に日本語学校が用意している寮に入居することが多い。そのため学生のステータスには**「寮生なのか否か」のフラグ**が必要。

### 規模

- 管理物件数: 約 **27件**

### 学生との紐付け

寮のテーブルには最低限「寮の名前」が必要。学生とJOINするには、**学生がどこの物件に住んでいるか**の情報が学生名簿側に必要。

::: warning 学生の異動に注意
学生が寮を引っ越したり変更するケースが頻繁に発生する。履歴管理の仕組みが必要。
:::

### 寮のマスターデータ

| 項目 | 内容 |
|------|------|
| 寮の名前 | 物件の識別名 |
| 住所 | 所在地 |
| 家賃 | 会社としての支出（販管費としての家賃） |
| 水光熱費 | 月次で発生する費用 |

#### ライフライン契約情報

物件ごとに以下の契約情報をマスターデータとして管理する必要がある。

| ライフライン | 管理項目 |
|------------|---------|
| ガス | 契約先、契約番号 |
| 水道 | 契約先、契約番号 |
| 電気 | 契約先、契約番号 |

### 寮の会計テーブル（月次）

寮に関する月次支出が発生するため、会計テーブルが必要。

| 支出科目 | 頻度 | 備考 |
|---------|------|------|
| 家賃 | 月次 | 不動産オーナーへの支払い |
| 電気代 | 月次 | 物件ごとに発生 |
| ガス代 | 月次 | 物件ごとに発生 |
| 水道代 | 月次 | 物件ごとに発生 |

### 課題

- **表記ゆれ**: 同一物件でもシートや書類によって名称が異なるケースがある。正規化ルールの整備が必要
- **Box vs Google Drive 照合**: 物件関連の書類が Box と Google Drive の両方に分散しており、どちらが最新か、どちらに何があるかの照合が必要

## WiFi 管理

### 規模

- デバイス数: **38台**
- 配置場所: **44箇所**
- 契約数: **22契約**（FREEDiVE社）

### データ整備状況

- 正規化DBを構築済み
- IMEI一覧を構築済み
- デバイスと配置場所の紐付け管理が可能な状態

## 水光熱費管理

### 利用サービス

- サービス名: **暮らしてぷこ**（TEPCO）
- 各物件の電気・ガス・水道の利用料金を一元管理
