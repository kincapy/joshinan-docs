# 共通 — 設計原則・技術スタック

## 設計原則

| # | 原則 | 説明 |
|---|------|------|
| 1 | Single Source of Truth | データの正規化と一元管理。同じ情報を複数箇所に持たない |
| 2 | 非エンジニアが運用できる | 専門知識なしで日常業務を回せるUI・ワークフロー |
| 3 | 段階的に育てる | 一度に全機能を作らず、優先度の高いものから順に実装 |
| 4 | データ入力は形式を問わない | Excel・CSV・手入力・API連携など、現場に合った方法を許容 |
| 5 | ローカルに依存しない | クラウドベースで、どの端末からでもアクセス可能 |
| 6 | ログ・バリデーション・監査証跡 | 誰が・いつ・何を変更したかを記録し、データの信頼性を担保 |
| 7 | 機能単位で設計・レビュー | 機能ごとに独立して開発・テスト・デプロイできる構成 |

## 技術スタック

### 現在（Phase 1〜5）

```
Excel + Box MCP + Google Drive + Claude.ai + openpyxl/pandas
```

- **Excel**: 統合版マスターデータの管理・帳票出力
- **Box MCP**: ファイル管理・共有・MCP連携
- **Google Drive**: 一部ドキュメントの共有
- **Claude.ai**: プロンプトベースでのデータ処理・変換
- **openpyxl / pandas**: Excelファイルの自動処理スクリプト

### 将来（Phase 6〜）

```
Supabase + Box + Vercel + Claude API + Box API
```

- **Supabase**: PostgreSQLデータベース + 認証 + リアルタイム
- **Box**: ファイルストレージ・ドキュメント管理
- **Vercel**: フロントエンド（Next.js）のホスティング
- **Claude API**: AI処理のバックエンド統合
- **Box API**: ファイル操作の自動化
